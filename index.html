Métodos de Clase y Dunders

Métodos de Instancia

Son los métodos más comunes. Operan sobre una instancia específica (self).

class Perro:
    def __init__(self, nombre):
        self.nombre = nombre # Atributo de instancia
    
    def ladrar(self): # Método de instancia
        print(f"{self.nombre} dice ¡Guau!")
2 / 15

Métodos de Clase (@classmethod)

Operan sobre la clase en sí (cls), no sobre una instancia. Se marcan con el decorador `@classmethod`.

class Perro:
    patas = 4 # Atributo de clase

    @classmethod
    def get_numero_patas(cls):
        return cls.patas
3 / 15

Uso de @classmethod

Se pueden llamar directamente desde la clase, sin necesidad de crear un objeto.

print(Perro.get_numero_patas()) # 4
4 / 15

Métodos Estáticos (@staticmethod)

Son funciones normales agrupadas dentro de una clase por lógica, pero no acceden ni a la clase (cls) ni a la instancia (self).

class Calculadora:
    @staticmethod
    def sumar(a, b):
        return a + b

Métodos Mágicos o "Dunder"

Métodos especiales con nombres que empiezan y terminan con doble guion bajo (Double Under). Permiten que tus objetos usen operadores de Python.

6 / 15

Dunder: __init__(self, ...)

El constructor. Se llama automáticamente al crear un objeto. Lo hemos usado para inicializar atributos.

class Libro:
    def __init__(self, titulo):
        self.titulo = titulo
7 / 15

Dunder: __str__(self)

Define la representación "amigable" de un objeto como string. Es lo que se muestra al usar print().

class Libro:
    def __init__(self, titulo):
        self.titulo = titulo
    
    def __str__(self):
        return f"Libro: '{self.titulo}'"

libro_py = Libro("Python 101")
print(libro_py) # Libro: 'Python 101'
8 / 15

Dunder: __repr__(self)

Define la representación "oficial" o de depuración de un objeto. Debe ser inequívoca.

class Libro:
    # ... __init__ y __str__ ...
    def __repr__(self):
        return f"Libro(titulo='{self.titulo}')"

# En una consola interactiva:
# >>> libro_py
# Libro(titulo='Python 101')
9 / 15

Dunder: __len__(self)

Permite que la función len() funcione con tus objetos.

class Estanteria:
    def __init__(self):
        self.libros = []
    
    def __len__(self):
        return len(self.libros)

est = Estanteria()
est.libros.append("Libro A")
print(len(est)) # 1

Dunders de Comparación

Permiten usar operadores como ==, <, >=, etc.

# __eq__(self, other) -> para ==
# __lt__(self, other) -> para <
# __gt__(self, other) -> para >
# ... y otros ...
11 / 15

Ejemplo con __eq__

Define cómo se comparan dos objetos de tu clase.

class Punto:
    def __init__(self, x, y):
        self.x, self.y = x, y
        
    def __eq__(self, otro):
        return self.x == otro.x and self.y == otro.y

p1 = Punto(1, 2)
p2 = Punto(1, 2)
print(p1 == p2) # True (Sin __eq__ sería False)

Dunders Aritméticos

Permiten que tus objetos usen operadores matemáticos como +, -, *.

# __add__(self, other) -> para +
# __sub__(self, other) -> para -
# __mul__(self, other) -> para *
# ... y otros ...
13 / 15

Ejemplo con __add__

Define qué sucede cuando sumas dos objetos de tu clase.

class Punto:
    # ... __init__ y __eq__ ...
    def __add__(self, otro):
        nuevo_x = self.x + otro.x
        nuevo_y = self.y + otro.y
        return Punto(nuevo_x, nuevo_y)

p1 = Punto(1, 2)
p2 = Punto(3, 4)
p3 = p1 + p2
print(f"({p3.x}, {p3.y})") # (4, 6)

Conclusión

Los métodos Dunder son el "pegamento" que permite que tus objetos se comporten como los tipos de datos nativos de Python, haciendo tu código más intuitivo y "Pythónico".
